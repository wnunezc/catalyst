# Phase 2 Source Operational Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Separate framework source code from operational drift while preserving the working `/demo-ui` Inspinia baseline, institutional color skins, and local WSDD development behavior.

**Architecture:** Treat this as a staged cleanup, not a bulk delete. First create machine-verifiable inventories, then repair references to canonical paths, then remove only assets/configs proven unused by static analysis and runtime browser checks. Demo UI remains the canonical public route; legacy `MigrationUi` naming is renamed only in a controlled pass with route parity checks.

**Tech Stack:** PHP 8.4, Bootstrap 5.3.8, Inspinia static assets, WSDD/Docker, PowerShell, Git, Catalyst CLI.
